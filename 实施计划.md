## 项目概述

基于 WebGPU 的 Rust 渲染引擎库，实现 DOM 操作和 CSS 布局，无 JavaScript 运行时。

### 架构决策

**WebGPU vs OpenGL**
- ✅ WebGPU：现代 API、更好的并行计算支持、跨平台
- ❌ OpenGL：老旧 API、并行计算支持有限

**Compute Shader vs CPU 布局**
- ✅ Compute Shader：元素级并行、性能极高
- ❌ CPU：串行递归、性能瓶颈

**自研 vs Servo**
- ✅ 自研：完全可控、GPU 架构
- ❌ Servo：老旧架构、CPU 为主

## 风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|-----|------|------|---------|
| WebGPU 驱动兼容性 | 高 | 中 | 降级方案、兼容层 |
| CSS 兼容性 | 中 | 高 | 渐进支持、明确范围 |
| 性能不达标 | 高 | 低 | 性能分析、优化迭代 |
| 开发周期超期 | 中 | 中 | 分阶段交付、MVP 优先 |
