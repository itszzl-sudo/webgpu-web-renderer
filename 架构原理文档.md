# WebGPU 渲染引擎架构原理文档

## 项目概述

**项目定位**：基于 WebGPU 的 Rust 渲染引擎库，提供 DOM 和 CSS 布局能力，无 JavaScript 运行时。

**核心特性**：
- 基于 WebGPU 原生 API 构建渲染管线
- 支持 DOM 操作和 CSS 布局规则
- GPU 并行计算布局，替代传统 CPU 递归重排
- 外部程序通过 bridge.rs 接口操作引擎

### 核心组件

#### 1. bridge.rs 桥接接口层

**职责**：提供外部程序操作渲染引擎的接口

**核心类型**：
- `WebNativeBridge` trait：定义所有操作接口
- `LayoutRect`：元素布局矩形
- `LayoutNode`：布局节点信息
- `Color`：颜色表示

**主要接口分类**：

**DOM 读写**：
- `set_html(html)` - 设置页面 HTML
- `query(selector)` - 查找元素
- `get_attr(node_id, name)` - 获取属性
- `set_attr(node_id, name, value)` - 设置属性
- `text(node_id)` - 获取文本内容

**CSS 操作**：
- `set_css(css_text)` - 添加 CSS 规则
- `set_style(selector, property, value)` - 设置元素样式
- `clear_css()` - 清除自定义 CSS

**布局查询**：
- `get_rect(selector)` - 获取元素位置
- `all_rects()` - 获取所有布局节点
- `hit_test(x, y)` - 点击测试

**渲染控制**：
- `render()` - 渲染当前页面，返回 PNG 字节

**事件处理**：
- `on_click(selector, handler)` - 绑定点击事件
- `handle_click(x, y)` - 处理鼠标点击

**网络请求**：
- `navigate(url)` - 导航到 URL
- `http_get(url)` - HTTP GET 请求
- `http_post(url, body, content_type)` - HTTP POST 请求

#### 2. DOM + CSS 解析层

**职责**：解析 HTML/CSS，构建 DOM 树，计算样式

**核心数据结构**：
```rust
struct DomNode {
    id: usize,
    tag_name: String,
    attributes: HashMap<String, String>,
    children: Vec<usize>,
    parent: Option<usize>,
    text_content: Option<String>,
}

struct StyleRule {
    selector: String,
    declarations: Vec<(String, String)>,
}
```

**功能模块**：
- HTML 解析器：将 HTML 文本解析为 DOM 树
- CSS 解析器：解析 CSS 规则
- 样式匹配器：将 CSS 规则匹配到 DOM 节点
- 样式计算器：计算每个节点的最终样式

#### 3. GPU 布局计算层

**职责**：使用 WebGPU Compute Shader 并行计算布局

**核心数据结构** (WGSL)：
```wgsl
struct LayoutItem {
    size: vec2f;        // 元素尺寸
    margin: vec4f;      // 外边距
    padding: vec4f;     // 内边距
    pos: vec2f;         // 元素位置
    flowType: u32;      // 0文档流 1绝对定位 2Flex
    weight: f32;        // flex 权重
    zIndex: f32;        // 层级
    texIdx: u32;        // 图文纹理索引
    isValid: u32;       // 元素是否启用
    isHide: u32;        // 是否隐藏
};

struct LayoutEnv {
    viewSize: vec2f;    // 视口尺寸
    flowDir: u32;       // 流向
    gap: vec2f;         // 间距
};
```

**布局模式映射**：

| WebGPU 物理字段 | CSS 属性 |
|----------------|---------|
| pos | left / top / transform |
| size | width / height |
| margin | margin |
| padding | padding |
| weight | flex-grow / flex-shrink |
| flowType | display (block, flex, absolute) |
| zIndex | z-index |
| isHide | display: none / visibility: hidden |

**布局流程**：
1. 将所有 DOM 元素转为 LayoutItem 存入 GPU 显存
2. Compute Shader 并行计算每个元素的位置
3. 根据不同 flowType 应用不同的布局规则：
   - 文档流：递推上一元素底部坐标
   - 绝对定位：直接使用自定义坐标
   - Flex：按权重分配空间
   - Grid：锁定到固定网格位置
4. 处理碰撞检测和避让
5. 输出最终布局坐标

#### 4. WebGPU 渲染管线

**职责**：根据布局结果进行渲染

**渲染流程**：

1. **纹理准备**：
   - 图片纹理：上传图片到 GPU
   - 文字纹理：SDF 字体图集

2. **顶点着色器**：
   - 读取 LayoutItem 获取元素位置和尺寸
   - 生成对应的顶点坐标
   - 计算 UV 坐标

3. **片段着色器**：
   - 根据 texIdx 选择对应的纹理
   - 采样纹理颜色
   - 应用背景色
   - 处理透明度和混合

4. **后处理**（可选）：
   - 模糊、锐化等效果
   - 色阶调整

## 关键技术特性

### 1. GPU 并行布局

**传统方式**：
- CPU 多线程树遍历递归求解
- 子树任务级并行
- 存在递归重排开销

**本方案**：
- GPU Compute Shader 元素级全并行
- 上万元素同时独立运算
- 无递归重排开销

### 2. CSS 等价物理布局

使用刚体物理模拟实现 CSS 布局效果：

| 布局效果 | 物理模拟 |
|---------|---------|
| 文档流 | 全局场力流向 |
| 绝对定位 | 自由无约束运动 |
| Flex 布局 | 弹簧约束 |
| Grid 布局 | 刚性卡槽约束 |
| 自适应环绕 | 碰撞避让 |
| 平滑过渡 | 惯性运动 + 阻尼 |

### 3. 增量更新机制

**读写分离**：
- 读样式/位置：优先读取 CPU 本地缓存
- 写样式：存入修改队列，帧末批量提交

**防抖合并**：
- 高频修改合并为单次 GPU 提交
- 消除布局抖动

**增量更新**：
- 仅推送变更字段
- 降低显存 IO 开销

## 性能优势

### 对比传统浏览器

| 指标 | 传统浏览器 | WebGPU 引擎 |
|-----|-----------|------------|
| 布局计算 | CPU 串行递归 | GPU 并行计算 |
| 并行粒度 | 子树任务级 | 元素级 |
| 重排开销 | 高 | 极低 |
| 动态修改 | 触发重排重绘 | 仅帧级数据同步 |
| JS 运行时 | 必需 | 无 |

### 核心优势

1. **性能极致优化**：布局从 CPU 串行递归 → GPU 数万线程并行
2. **架构极简**：文字、图片、容器共用一套渲染与布局体系
3. **动态修改开销极低**：通过 bridge.rs 操作不再产生昂贵重排重绘
4. **内核高度自研可控**：脱离 Chromium、Servo 传统老旧布局管线

## 使用示例

```rust
use webgpu_renderer::WebNativeBridge;

// 创建渲染引擎实例
let mut engine = MyWebGpuRenderer::new(1920, 1080);

// 设置 HTML
engine.set_html(r#"
    <div id="container">
        <h1>Hello World</h1>
        <p>This is a WebGPU rendering engine</p>
    </div>
"#);

// 设置 CSS
engine.set_css(r#"
    #container {
        padding: 20px;
        background: #f0f0f0;
    }
    h1 {
        color: #ff0000;
        font-size: 32px;
    }
    p {
        color: #333;
        font-size: 16px;
    }
"#);

// 查询元素
let container = engine.query("#container").unwrap();
engine.set_attr(container, "class", "main-container");

// 设置样式
engine.set_style("h1", "margin", "10px 0");

// 渲染
let png_data = engine.render();
std::fs::write("output.png", png_data).unwrap();
```
